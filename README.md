WINDOWS MODULAR TOOLKIT
Framework Edition 2.0
==================================

A USB-friendly framework for organizing, managing, validating, and running modular tools.

Core idea:
- The framework manages itself.
- Users manage the content.

Framework components are protected and should not be renamed, moved, hidden, or deleted through the toolkit.
User modules, categories, installers, websites, and folders are user-managed content.

============================================================
 WINDOWS MODULAR TOOLKIT
============================================================
[S] Search Center
    Find tools, modules, documentation, repository items, workspace links, and framework resources.
[L] Launch Center
    Browse and launch toolkit modules, software deployments, installers, and utilities.

------------------------------------------------------------

[W] Workspace Center
    Open personal toolkit shortcuts and linked items. Workspace links by default; it does not move/copy modules.
[D] Dashboard Center
    View toolkit status, health summaries, statistics, recommendations, and recent activity.

------------------------------------------------------------

[F] Framework Center
    Maintain, repair, audit, and configure the framework itself.
[M] Module Tool Manager
    Create, validate, package, repair, and maintain modules.
[R] Repository Manager
    Scan Incoming, organize Repository items, and manage software deployments.
[T] Tool Manager
    Organize, rename, move, clone, hide, remove, scan, and manage existing tools.

------------------------------------------------------------

[H] Help Center
    Documentation, tutorials, troubleshooting, standards, architecture, and guidance.
[Q] Quit

Toolkit Management contains framework tools used to create, organize, validate, repair, automate, update, and maintain the toolkit.
Browse Modules contains user tools and modular content.

Navigation Standard
-------------------
B = Back one level from any submenu.
Q = Quit is only shown and used on the Main Menu.


Framework vs Content

Framework Center manages the toolkit itself.

Module Tool Manager creates and maintains modules.

Repository Manager manages files, software, installers, documents, archives, disk images, and imported content.

Tool Manager manages existing user tools and modules.

Most users will spend their time in:

* Search Center
* Launch Center
* Workspace Center
* Repository Manager

Framework Center is typically used only when configuring, repairing, validating, or extending the toolkit.
