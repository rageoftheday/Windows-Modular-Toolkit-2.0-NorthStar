# Lessons Learned

## Preserve Why, Not Just What
The framework should preserve the reasoning behind decisions, not only the code.

## Validate Before Executing
Dry-run and validation capabilities exist because broken metadata, paths, and scripts were discovered too late in earlier builds.

## Categories Must Be Managed
Categories are required to prevent unorganized tools and long-term clutter. There is no Uncategorized category.

## Framework Updates Never Touch Modules
User-created tools are content. The framework must not overwrite or delete them during updates or repairs.

## Debugging Needs Traceability
When category behavior became confusing, diagnostic logs quickly revealed what was actually happening. Framework debug capability is valuable and should remain available when needed.
