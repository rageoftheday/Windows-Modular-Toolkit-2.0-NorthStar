# Framework Usability Polish

## Startup behavior

Healthy startup should not stop the user. Framework checks run silently and main status is shown on the main menu.

Warning/error screens should appear only when action may be needed.

## Purpose text

Show purpose text for non-obvious features, managers, framework centers, modules, workspace sections, and workflows. Do not show purpose text for obvious navigation actions such as Back, Quit, Exit, Next, or Previous.
